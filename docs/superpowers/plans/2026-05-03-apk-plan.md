# 指数观察 APK 打包 — 实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将现有 Web 原型打包为安卓 APK，手机安装后独立联网更新数据

**Architecture:** GitHub Actions 每天自动拉取全部数据（S&P 500、VIX、沪深300点位、沪深300 PE），发布到 GitHub Pages 的 JSON 文件。APK 前端只从 GitHub Pages 读取数据，无需后端服务。

**Tech Stack:** Capacitor（打包）、GitHub Actions（数据更新）、原生 HTML/CSS/JS（界面）

## 第二步：GitHub Actions 自动数据更新

### 任务 3：创建数据拉取 Python 脚本

**Files:**
- 修改: `scripts/fetch_hs300.py`（改名为 `scripts/fetch_all.py`，改为拉取全部数据）

- [ ] **操作 1：将现有 PE 脚本改为拉全部数据**

当前 `scripts/fetch_hs300.py` 只拉沪深300 PE。需要扩展它：同时拉 S&P 500、VIX、沪深300 点位，输出两份 JSON。

新建 `scripts/fetch_all.py`，将 S&P 500 和 VIX 的数据也通过 FRED API 拉取（用 Python 的 requests 库），沪深300 指数点位用东方财富 API，PE 继续用 AKShare。最终输出：
  - `data/cache/spx-vix.json`
  - `data/cache/hs300-valuation.json`

### 任务 4：创建 GitHub Actions 工作流

**Files:**
- 创建: `.github/workflows/update-data.yml`

- [ ] **操作 1：创建工作流配置**

```yaml
name: 每日更新指数数据

on:
  schedule:
    - cron: '30 22 * * *'  # 北京时间早上6:30（UTC 22:30）
  workflow_dispatch:  # 允许手动触发

jobs:
  update:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4
      
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      
      - name: 安装依赖
        run: pip install akshare requests
      
      - name: 拉取数据
        run: python scripts/fetch_all.py
      
      - name: 复制到发布目录
        run: |
          mkdir -p public_data
          cp data/cache/spx-vix.json public_data/
          cp data/cache/hs300-valuation.json public_data/
      
      - name: 发布到 GitHub Pages
        uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public_data
          publish_branch: gh-pages
```

- [ ] **操作 2：提交并推送**
```bash
git add -A
git commit -m "添加 GitHub Actions 自动数据更新流水线"
git push
```

## 第三步：Capacitor APK 打包

### 任务 6：安装必要工具

**操作人：我（命令行操作）**

- [ ] **操作 1：安装 Node.js 依赖和 Capacitor CLI**

项目已有 Node.js，直接安装 Capacitor：
```bash
cd "C:/Users/YM/Documents/Codex/2026-05-03/app-app-app-https-www-macromicro"
npm init -y  # 如果已有 package.json 则跳过
npm install @capacitor/core @capacitor/cli @capacitor/android
```

- [ ] **操作 2：初始化 Capacitor**

```bash
npx cap init "指数观察" "com.indexwatch.app" --web-dir=public
```

参数含义：
- `指数观察` = APP 名称
- `com.indexwatch.app` = APP 内部标识（安卓用）
- `--web-dir=public` = 界面文件所在目录

- [ ] **操作 3：添加安卓平台**
```bash
npx cap add android
```

这会创建一个 `android/` 目录，里面是 Android Studio 项目。

### 任务 7：安装 Java 和 Android SDK

**操作人：用户需要在自己的电脑上操作部分**

> 打包 APK 需要 Java 17+ 和 Android 构建工具。这些是比较大的软件（几个 GB），需要用户在自己的环境里安装。

- [ ] **操作 1：检查 Java 是否已安装**
```bash
java -version 2>&1
```
如果显示 `java version "17"` 或更高，直接跳到任务 8。如果显示找不到命令或版本太低，继续下一步。

- [ ] **操作 2：安装 Java 17**
  如果没装 Java，让用户在浏览器打开 `https://adoptium.net/download/` 下载安装 Java 17（Windows x64 msi 安装包），安装后重新打开终端验证。

- [ ] **操作 3：安装 Android Studio**
  - 下载：`https://developer.android.com/studio`
  - 安装后首次打开，按默认选项完成初始设置（会下载 Android SDK）
  - 记下 Android SDK 的安装路径（通常是 `C:\Users\<用户名>\AppData\Local\Android\Sdk`）

### 任务 8：构建 APK

**操作人：我（命令行操作）**

- [ ] **操作 1：配置 Android 项目**

在 `android/` 目录下需要做几个配置：
- 修改 `android/app/src/main/res/values/strings.xml`：设置 APP 名称为"指数观察"
- 修改 `android/app/src/main/AndroidManifest.xml`：确保有网络权限 `<uses-permission android:name="android.permission.INTERNET"/>`

- [ ] **操作 2：同步前端代码到安卓项目**
```bash
npx cap sync android
```

- [ ] **操作 3：构建 APK**
```bash
cd android
./gradlew assembleDebug
```
成功后 APK 文件位于：
`android/app/build/outputs/apk/debug/app-debug.apk`

- [ ] **操作 4：安装到手机**
  将 APK 文件传到手机，在手机上点击安装。
  （首次安装未知来源 APK 需要在手机设置里允许）

## 限制说明

- **数据更新频率**：免费 GitHub Actions 每月 2000 分钟，每天跑一次完全够用
- **首次数据**：GitHub Actions 工作流推送到仓库后会自动运行一次（在设置完 Pages 之前）
- **数据延迟**：PE 数据每天更新一次（交易日才有变化），S&P 500 和 VIX 每天更新一次
