# 项目工作守则

## 用户背景

- 零编程经验，完全不会写代码
- 不熟悉终端/命令行操作
- 不熟悉英文技术术语
- 目标是：通过对话讨论并开发设计程序及APP

## 沟通规则

- 所有回复和解释使用中文
- 涉及英文技术术语时附带中文说明
- 涉及终端命令时，解释清楚每一步在做什么
- 重大操作（安装软件、删除文件、系统级修改等）务必先解释再请求确认

## 工作流程

每项任务必须遵循：先讨论确认方向 → 再制定计划 → 再逐步实现

- 收到需求后，先和用户讨论方案选项（列出选择 + 各自的优缺点），不要直接动手
- 任何时候存在官方与非官方选项之分，必须先列出选项标注来源性质，等用户选择
- 确认方向后用 planning 流程写计划，用户批准后再动手
- 实现过程中，优先小步修改，不要一次性重写整个项目

## 项目信息

- 名称：指数观察 APP（Index Watch）
- 当前阶段：Web 原型 → 准备打包为 APK
- 技术栈：Node.js 原生 HTTP 服务 + 原生 HTML/CSS/JS + Canvas 绘图
- 数据源：FRED（S&P 500 + VIX）、AKShare/Python（沪深300 PE）
- 本地访问：http://localhost:4173
- 启动方式：npm start
