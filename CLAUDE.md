# 项目协作守则

## 1. 读取用户背景

每次会话开始时，先读取用户的个人记忆：**"用户背景与能力"**

用户零编程经验，不熟悉终端命令和英文技术术语。所有沟通使用中文，重大操作先解释再请求确认。

## 2. 开发流程：先规划再动手

收到需求后，不要直接写代码。按以下顺序：

1. **brainstorming** — 讨论方案方向，列出选项和优缺点
2. **writing-plans** — 写实现计划，用户批准后再动手
3. 逐步实现，小步修改，不一次性重写整个项目

遇到不确定的问题时，先调用相关 skill 看有没有帮助文档，不要自己硬猜。

## 3. 遇到难题先查技能

项目执行过程中遇到解决不了的问题时：
- 先用 brainstorming 梳理问题
- 再检查已安装的 skill 列表，看有没有相关技能能帮忙
- 不要在不熟悉的领域直接写代码

## 4. 先出 Web 原型确认样式

用户不会画设计图，所以需求定稿后的流程是：

1. 先开发 Web APP 原型（能在浏览器打开看到界面效果）
2. 用户在浏览器里确认样式和数据没问题
3. 确认后再继续开发最终形态（APK、桌面软件等）

Web 原型技术选型：Node.js 原生 HTTP + 原生 HTML/CSS/JS，界面要能在浏览器直接看到。
